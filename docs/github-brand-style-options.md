# GitHub Brand Style Options

This note captures three GitHub-specific personal brand directions for Caprika as an agent-native developer.

## Recommended: Technical Editorial

Use `assets/banner-technical-editorial.svg`.

Why it fits:

- It keeps the name fully visible and readable on GitHub.
- It feels technical without becoming a generic hacker terminal.
- It uses the right-side space for a runtime map instead of accidental cropping.
- It connects directly to the current positioning: agent skills, local runtime, filesystem artifacts, reviewable software.

## Alternative: Agent Terminal

Use `assets/banner-agent-terminal.svg`.

Why it fits:

- It is direct and developer-native.
- It makes the profile feel like a working CLI surface.
- It is strong for coding-agent identity, but slightly narrower and more hacker-coded than the recommended option.

## Alternative: Open-Source Maintainer

Use `assets/banner-oss-maintainer.svg`.

Why it fits:

- It looks closest to GitHub's own visual language.
- It emphasizes review, artifacts, and open-source maintenance.
- It is trustworthy and quiet, but less distinctive than the recommended option.

## Decision

Use Technical Editorial as the GitHub default. It is the best balance between personal brand, technical credibility, Open Design positioning, and GitHub readability.
